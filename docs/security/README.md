# Security Documentation

Internal security documentation for StrideAlytics.

## Contents

- `SECURITY.md` (root) — Public-facing security policy, vulnerability reporting, supported versions.
- `hardening.md` — Implementation hardening checklist (secrets rotation, CI linting, RLS verification).
- `architecture-decisions.md` — Security-focused architecture decision records (ADRs).
- `audit-log.md` — Security audit log template and instructions.
- `incident-response.md` — Incident response playbook.

## Scope

This folder is for internal/team use. The root `SECURITY.md` is the public entry point for external researchers.
